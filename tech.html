    • |2| O Diskpart permite formatar em sistemas de ficheiros NTFS, FAT32 e exFAT. O NTFS é o mais falado no Windows, então vamos ver qual é a diferença entre FAT32 e exFAT, os sistemas de ficheiros que rodam noutros sistemas operacionais como Linux e macOS? (PT)

    O sistema de arquivos FAT32 é mais antigo do que o exFAT. O FAT32 foi introduzido no Windows 95 para substituir o ainda mais antigo FAT. Ele é amplamente compatível e é considerado o formato padrão para dispositivos de armazenamento móveis, como pen drives e cartões de memória. Por outro lado, o exFAT foi projetado como sucessor do FAT32, focado em compatibilidade e na capacidade de gravar arquivos grandes. O FAT32 é mais antigo e tem a maior compatibilidade entre os sistemas operacionais e dispositivos. Ele pode ser lido e gravado por quase todos os sistemas operacionais e a maioria dos dispositivos com uma porta USB. Por outro lado, o exFAT também é compatível com muitos dispositivos, mas não todos. O FAT32 tem limitações significativas no tamanho do arquivo e da partição. Ele não suporta arquivos maiores do que 4GB e partições maiores do que 8TB. Em contraste, o exFAT permite tamanhos de arquivo e partição muito maiores. Teoricamente, o exFAT pode suportar arquivos de até 16 exbibytes (1EiB ~= 1,152,921,504,606,846,976 bytes) e partições de até 128 pebibytes (1PiB ~= 1,125,899,906,842,624 bytes). O FAT32 é menos eficiente e tem menos recursos em comparação com o exFAT1. O exFAT possui recursos avançados semelhantes ao NTFS, como a capacidade de recuperar arquivos e suporte para criptografia. O FAT32 é geralmente usado para dispositivos de armazenamento removíveis, como pen drives e cartões SD. O exFAT, por outro lado, é ideal para unidades externas destinadas a vários sistemas operacionais e é melhor para tamanhos de arquivo e partições maiores.

    |2| The Diskpart allows formatting in NTFS, FAT32, and exFAT file systems. NTFS is the most commonly used in Windows, so let’s see what the difference is between FAT32 and exFAT, the file systems that run on other operating systems like Linux and macOS? (EN)

    The FAT32 file system is older than exFAT. FAT32 was introduced in Windows 95 to replace the even older FAT. It is widely compatible and considered the standard format for mobile storage devices such as USB drives and memory cards. On the other hand, exFAT was designed as the successor to FAT32, focusing on compatibility and the ability to handle large files. FAT32 is older and has the broadest compatibility across operating systems and devices. It can be read and written by almost all operating systems and most devices with a USB port. Conversely, exFAT is also compatible with many devices, but not all. FAT32 has significant limitations on file and partition size. It does not support files larger than 4GB and partitions larger than 8TB. In contrast, exFAT allows much larger file and partition sizes. Theoretically, exFAT can support files up to 16 exbibytes (1EiB ~= 1,152,921,504,606,846,976 bytes) and partitions up to 128 pebibytes (1PiB ~= 1,125,899,906,842,624 bytes). FAT32 is less efficient and has fewer features compared to exFAT1. ExFAT includes advanced features similar to NTFS, such as file recovery capability and encryption support. FAT32 is generally used for removable storage devices like USB drives and SD cards. ExFAT, on the other hand, is ideal for external drives intended for multiple operating systems and is better suited for larger file sizes and partitions.

    |3| Os Quatro níveis de Controlo de Conta de Utilizador (UAC) no Windows: (PT)

      (1) Sempre notificar: O UAC notifica-o antes de os programas efetuarem alterações que requerem privilégios de administrador. (2) Notificar-me apenas quando os programas/aplicações tentarem efetuar alterações no meu computador: O UAC notifica-o quando os programas tentarem efetuar alterações ou instalar software no seu computador. (3) Notificar-me apenas quando os programas/aplicações tentarem efetuar alterações no meu computador (sem escurecer o ecrã): Semelhante ao modo anterior, mas sem o efeito de escurecimento do ecrã. (4) Nunca notificar: O UAC está desativado e não o notificará sobre alterações que requerem privilégios de administrador.

    |3| The Four User Account Control (UAC) levels in Windows: (EN)

      (1) Always notify: UAC notifies you before programs make changes that require administrator privileges. (2) Notify me only when programs/apps try to make changes to my computer: UAC notifies you when programs attempt to make changes or install software on your computer. (3) Notify me only when programs/apps try to make changes to my computer (without dimming the screen): Similar to the previous mode, but without the screen dimming effect. (4) Never notify: UAC is disabled and won’t notify you about changes that require administrator privileges.
